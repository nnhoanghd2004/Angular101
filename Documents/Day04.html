Buổi 4: Cách tạo và sử dụng Service

Trong Angular, Component là một thành phần cơ bản của ứng dụng. Component có nhiệm vụ hiển thị dữ liệu và xử lý logic cơ bản. Tuy nhiên, đối với những logic phức tạp hoặc business logic, việc đặt chúng trong Component sẽ khiến ứng dụng trở nên khó bảo trì và mở rộng.
Chính vì vậy, Angular cung cấp một lớp khác có tên là Service để xử lý các logic phức tạp hoặc business logic. Service có thể được sử dụng bởi nhiều Component khác nhau trong ứng dụng, giúp tăng tính tái sử dụng và bảo trì.

1. Service là gì?

Service là gì?

Service trong Angular là một lớp (class) được sử dụng để cung cấp các chức năng chung cho các thành phần trong ứng dụng. Các service có thể được sử dụng để truy cập dữ liệu, thực hiện các tác vụ, hoặc cung cấp các tính năng chung khác.
Việc sử dụng Service trong Angular mang lại một số lợi ích sau:
Tăng tính tái sử dụng: Các Service có thể được sử dụng bởi nhiều Component khác nhau trong ứng dụng, giúp giảm thiểu sự trùng lặp mã.
Tăng tính bảo trì: Các Service được đặt trong một lớp riêng biệt, giúp dễ dàng quản lý và bảo trì mã hơn.
Tăng tính testability: Các Service có thể được kiểm tra độc lập, giúp đảm bảo chất lượng của mã.

3. Cách sử dụng Service

Nếu các bạn chưa biết Dependency Injection (DI) là gì thì đọc tại Dependency Injection (Optional) cho các bạn muốn hiểu cách hoạt động của nó

Sử dụng Service

Giả sử trong product.service.ts có đoạn code như sau

export class ProductService { products = [ { id: 1, name: 'Product 1' }, { id: 2, name: 'Product 2' }, { id: 3, name: 'Product 3' }, { id: 4, name: 'Product 4' }, ]; getProducts() { return this.products; } removeProduct(id: number) { this.products = this.products.filter((product) => product.id !== id); } }

Ở đây:
ngOnInit: Khi mà component được khởi tạo thì hàm này sẽ được tự động gọi một lần. Tận dụng hàm này để lấy danh sách product và gán vào mảng products của Component để hiển thị ra
removeProduct: Hàm này mục đích chính là gọi lại hàm removeProduct bên ProductService. Vì vậy Component chỉ cần gọi hàm này trong khi Service sử lý logic.
Lưu ý: Trong hàm removeProduct phải gọi lại hàm getProducts thêm một lần nữa vì biến products của Service đang có giá trị khác với biến products của Component. Vì vậy sau khi xoá thì cần phải cập nhận lại giá trị

Dependency Injection (Optional)

Tổng quan cách hoạt động của Dependency Injection

Bạn quan tâm cách Dependency Injection hoạt động như nào sao? Trước hết ta cần biết Dependency Injection là gì.
Dependency Injection (DI) trong lập trình là một kỹ thuật cung cấp các đối trượng, module cần thiết cho một đối tượng cấp cao hơn, thay vì bản thân đối tượng cấp cao đó tự tạo chúng. Nói đơn giản hơn, thay vì một lớp tự tìm kiếm và tạo các đối tượng khác mà nó cần để hoạt động, các đối tượng đó được "tiêm" vào nó từ bên ngoài.
Ví dụ về 2 đoạn code Typescript sau
TypeScript
Copy
class HttpService { get(url: string) { return "Get" + url; } } class LoggerService { save(log: string) { console.log(log); } } class ProductService { _httpService: HttpService; _loggerService: LoggerService; // class ProductService coupling quá chặt vào class HttpService và class LoggerService // Đồng thời câu hỏi được đặt ra là class khác cũng muốn sử dụng // class HttpService và class LoggerService thì sao? Chả lẽ mỗi class sử dụng // đều dùng new class? constructor() { this._httpService = new HttpService(); this._loggerService = new LoggerService(); } } const productService = new ProductService();
Without Depenency Injection
​
TypeScript
Copy
class HttpService { get(url: string) { return "Get" + url; } } class LoggerService { save(log: string) { console.log(log); } } class ProductService { _httpService: HttpService; _loggerService: LoggerService; // Lúc này class ProductService phụ thuộc vào instance của class HttpService // và class LoggerService. Thì luúc này ta chỉ "tiêm" bằng cách truyền instance // vào constructor. Như thế thì nếu có class khác cũng cần những service này // thì ta chỉ việc đem tiêm bình thường thôi constructor(httpService: HttpService, loggerService: LoggerService) { this._httpService = httpService; this._loggerService = loggerService; } } const httpService = new HttpService(); const loggerService = new LoggerService(); const productService = new ProductService(httpService, loggerService);
With Dependency Injection
​
Nhìn đoạn code bên dưới ta có thể dễ thấy rằng là ta không cần tạo class trực tiếp bên trong construtor mà chỉ cần việc đem instance vô là được
TypeScript
Copy
class HttpService { get(url: string) { return "Get" + url; } } class LoggerService { save(log: string) { console.log(log); } } class ProductService { _httpService: HttpService; _loggerService: LoggerService; constructor(httpService: HttpService, loggerService: LoggerService) { this._httpService = httpService; this._loggerService = loggerService; } } class UserService { _httpService: HttpService; constructor(httpService: HttpService) { this._httpService = httpService; } } class InterceptorService { _loggerService: LoggerService; constructor(loggerService: LoggerService) { this._loggerService = loggerService; } } const httpService = new HttpService(); const loggerService = new LoggerService(); const productService = new ProductService(httpService, loggerService); const userService = new UserService(httpService); const interceptorService = new InterceptorService(loggerService);
​

Dependency Injection trong Angular

Tương tự như Dependency Injection như bình thường thì Angular hỗ trợ việc tạo instance. Chúng ta chỉ cần tạo Service và “tiêm” nó vào Component là được (Không nhất thiết là Component, Các thứ khác như Service, Guard, Interceptor,… Đều tiêm được như bình thường)
Lưu ý:
Ở đây mình chỉ nói Constructor Injection, được xài nhiều nhất và được khuyến nghị nhất. Các bạn có thể xem các cách còn lại ở trên mạng nhé.
Cách tiêm ở trên là cách tối giản nhất chưa qua Dependency Inversion (một trong năm nguyên tắc SOLID). Vì ở trên vẫn còn phụ thuộc class trực tiếp thay vì interface
