Buổi 1: Cách cài đặt và làm việc với Angular

Trước khi sử dụng Angular hay những Frameworks và Libraries khác thì ta sử HTML, CSS, JS để xây dựng các trang web hoàn chỉnh. Vậy thì những Frameworks và Libraries khác nói chung và Angular nói riêng được sinh ra để xây dựng những trang web một cách nhanh chóng hơn cách truyền thống, giúp tăng hiệu xuất làm việc. Mặc dù vậy thì ta vẫn những kiến thức ấy để sử dụng một cách hiệu quả
HTML: Angular sử dụng HTML để định nghĩa giao diện của ứng dụng. Do đó, bạn cần hiểu các thẻ HTML cơ bản, cách sử dụng các thuộc tính HTML,...
CSS: Angular sử dụng CSS để định kiểu cho giao diện của ứng dụng. Do đó, bạn cần hiểu các khái niệm cơ bản về CSS, bao gồm các thuộc tính CSS, cách sử dụng các bộ tiền tố CSS,...
JavaScript (Typescript): Angular được viết bằng JavaScript (Typescript), vì vậy bạn cần có kiến thức cơ bản về ngôn ngữ này, bao gồm các khái niệm như biến, vòng lặp, hàm, đối tượng, sự kiện,...

1. Giới thiệu

Angular là một framework JavaScript mã nguồn mở được viết bằng TypeScript và được sử dụng để thiết kế giao diện web (front-end). Angular được xây dựng, phát triển từ năm 2009 và đang duy trì cho đến nay bởi Google. Đây được xem là framework front-end mạnh mẽ và chuyên dụng dành cho các lập trình viên sử dụng HTML, CSS và TypeScript.
Angular được ứng dụng rộng rãi để xây dựng các project Single Page Application (ứng dụng trang đơn). Phiên bản stable của Angular hiện tại là Angular 17 (phát hành vào ngày 14 tháng 2 năm 2023) với TypeScript 4.9 trở lên.
Xem thêm: https://stackoverflow.com/questions/21862054/single-page-application-advantages-and-disadvantages

3. Làm việc với Angular

Đây là Project Angular sau khi tạo xong

Okay, có vẻ rườm rà dành cho người mới bắt đầu nhỉ? Giờ bắt đầu phân tích Project

src: Codebase của các bạn
app: Nơi các bạn code ở thư mục này
app.component.html: File HTML. Nơi các bạn Code HTML
app.component.scss: File SCSS (SCSS tương tự như CSS nhưng mạnh hơn =)), nên thể sử dụng CSS trong SCSS )
app.component.spec.ts: File Test (Có thể skip cho tới khi đụng tới testing)
app.component.ts: File Typescript của component. Nơi các bạn đụng tới logic của code
assets: Nơi các bạn lưu trữ static resources (fonts, images,…)
index.html: file html root (TUYỆT ĐỐI KHÔNG ĐỤNG TỚI FILE NÀY)
main.ts: file khởi động (TUYỆT ĐỐI KHÔNG ĐỤNG TỚI FILE NÀY)
styles.scss: file css/scss root (Chỉ được đụng khi cần khai báo thư viện css, hoặc global variable)
.gitignore: Nơi khai báo các file không được tracked (các file .env, service accounts, …)
angular.json: Các thông số của workspace Angular, không cần đụng file này trừ khi build size cần được tăng
package.json: Nơi khai báo tên app, version và các thư viện được sử dụng
tsconfig.json: Nơi config typescript
